# 06_EVOLUTION_SPEC.md

> **ROLE**: Self-Improvement.
> **RESPONSIBILITY**: Code Mutation, Research, Fine-Tuning.

## 1. Prime Evolution Law
**Capability upgrades must be earned, affordable, testable, and reversible.**

## 2. The Paper-to-Code Loop (Cerebellum)
1.  **Read**: Ingest ArXiv paper.
2.  **Code**: Generate implementation.
3.  **Simulate**: Run in WorldModel.
4.  **Mutate**: Create Git Commit.
5.  **Verify**: Run Tests.
6.  **Merge**: If metrics improve & budget allows.

## 3. Self-Fine-Tuning
-   **Trigger**: Repeated failure / Economic opportunity.
-   **Scope**: Low-Rank Adapters (LoRA) only. No full fine-tuning.
-   **Lifecycle**: Adapters decay if unused.

## 4. Immune System
-   **Role**: Reject harmful mutations.
-   **Mechanism**: CI/CD pipeline + Invariant checks.
-   **Action**: Revert commit if regression detected.
